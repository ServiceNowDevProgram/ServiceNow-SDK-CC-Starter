# LIST_CONTROL

# ServiceNow LIST_CONTROL - API Reference

List Control (`sys_ui_list_control`) controls visibility and behavior of ServiceNow list UI elements (buttons, filters, links) based on roles or conditions.

**Table**: `sys_ui_list_control`
**API**: Record API from `@servicenow/sdk/core`

## Record API Parameters

| Property                  | Type             | Required | Default | Description                                                                                                                                                                                                                       |
| ------------------------- | ---------------- | -------- | ------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `$id`                     | `Now.ID[string]` | Yes      | -       | Unique identifier. Format: `Now.ID['value']`                                                                                                                                                                                      |
| `table`                   | `string`         | Yes      | -       | Must be `'sys_ui_list_control'`                                                                                                                                                                                                   |
| `name`                    | `TableName`      | Yes      | -       | Table name this list control applies to. Must be a valid table name.                                                                                                                                                              |
| `related_list`            | `String`         | No       | -       | table.field - Use when referencing a standard relationship (e.g., `incident.parent_incident`) REL:sys_id - Use when referencing a custom relationship by its sys_id(`REL:b9edf0ca0a0a0b010035de2d6b579a03`)                       |
| `label`                   | `String`         | No       | -       | Label to display for this list or related list. If not supplied, uses the default plural label for the table. Example: "Incidents" for incident table.                                                                            |
| `omit_new_button`         | Boolean          | No       | false   | Option to prevent the New button from displaying on this list. Clear the check box to display the New button.                                                                                                                     |
| `omit_edit_button`        | Boolean          | No       | true    | Option to prevent the Edit button from displaying on this list. Clear this check box to display the Edit button.                                                                                                                  |
| `omit_links`              | Boolean          | No       | false   | Option to hide links for fields that reference other files in this list. Leave this button unchecked to generate links for all reference fields.                                                                                  |
| `omit_drilldown_link`     | Boolean          | No       | false   | Option to disable the link to the record from the first column in list view. Users can still select the reference icon to access the record.                                                                                      |
| `omit_filters`            | Boolean          | No       | false   | Option to hide filters or breadcrumbs for this list. Clear this check box if you always want filters or breadcrumbs to be visible.                                                                                                |
| `omit_if_empty`           | Boolean          | No       | false   | Option to omit the **Related List** from the form entirely (no header) if there are no entries for the **Related List**.                                                                                                          |
| `omit_columns_if_empty`   | Boolean          | No       | false   | Option for a top-level list to omit the column headers AND filters/breadcrumbs for an empty column.                                                                                                                               |
| `new_roles`               | String[]         | No       | -       | User roles that can access the New button on this list. Leave the field blank to enable all users to access the New button. Array containing comma-separated string of role names. Example: ["admin, itil"]                       |
| `edit_roles`              | String[]         | No       | -       | User roles that can access the Edit button on this list. Leave the field blank to enable all users to access the Edit button. Array containing comma-separated string of role names. Example: ["admin, itil"]                     |
| `filter_roles`            | String[]         | No       | -       | User roles required to have the filter appear in the list. Leave the field blank to enable all users to access the filter. Array containing comma-separated string of role names. Example: ["admin, itil"]                        |
| `link_roles`              | String[]         | No       | -       | User roles required to have links generated for fields that reference other files. Leave the field blank to enable all users to access the links. Array containing comma-separated string of role names. Example: ["admin, itil"] |
| `new_condition`           | `Script`         | No       | -       | If condition evaluates true, omit New button. Use `Now.include()` for external scripts.                                                                                                                                           |
| `edit_condition`          | `Script`         | No       | -       | If condition evaluates true, omit Edit button. Use `Now.include()` for external scripts.                                                                                                                                          |
| `link_condition`          | `Script`         | No       | -       | If condition evaluates true, omit related links. Use `Now.include()` for external scripts.                                                                                                                                        |
| `empty_condition`         | `Script`         | No       | -       | If condition evaluates true, omit list header if there are no entries. Use `Now.include()` for external scripts.                                                                                                                  |
| `columns_condition`       | `Script`         | No       | -       | If condition evaluates true, omit column headings if there are no entries. Use `Now.include()` for external scripts.                                                                                                              |
| `filter_condition`        | `Script`         | No       | -       | If condition evaluates true, omit list filter options. Use `Now.include()` for external scripts.                                                                                                                                  |
| `omit_count`              | `Boolean`        | No       | false   | Remove pagination count.                                                                                                                                                                                                          |
| `omit_count_views`        | `Boolean`        | No       | false   | Prevent specific views from getting full record count                                                                                                                                                                             |
| `omit_count_views_list`   | List<String>     | No       | -       | List of sys_id of specific views in comma separated format where count will be removed, reference UI View( sys_id ) ex: 'f82b9231f3ba7210bd04a339f6968ce7,a84d5af5f3ba7210bd04a339f6968cfc'                                       |
| `omit_related_list_count` | `Boolean`        | No       | false   | Remove related list record count on form load within Workspace                                                                                                                                                                    |
| `hierarchical_lists`      | `Boolean`        | No       | false   | Enable hierarchical list display                                                                                                                                                                                                  |
| `disable_nlq`             | `Boolean`        | No       | false   | Disables Natural Language Query from the filter                                                                                                                                                                                   |
| `list_edit_type`          | `String`         | No       | -       | Type of list editing (inline editing) allowed to edit values directly in individual cells in a list. The options are: 'save_by_row', 'disabled', or do not add list_edit_type for default behavior.                               |
| `list_edit_ref_qual_tag`  | `String`         | No       | -       | Unique tag that is sent to a reference qualifier as the script variable listEditRefQualTag. Enter an arbitrary string of letters or numbers.                                                                                      |
| `list_edit_insert_row`    | `Boolean`        | No       | false   | Create records in list view. When it is enabled, an empty row appears at the bottom of the list.                                                                                                                                  |
| `active`                  | `boolean`        | No       | true    | Must be true for list control to take effect                                                                                                                                                                                      |
